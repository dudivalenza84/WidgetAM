# 2026-06-22 · #02 — Recuperar ambiente e reconstrução de features (widget + .dmg)

**Status:** parcial (apenas Etapa 0 concluída nesta sessão)

## Objetivo

Recuperar o ambiente quebrado pelo pnpm e evoluir o MacMediaWidget para um app de
verdade: integração funcional com o Amazon Music rodando como PWA, comportamento de
widget de mesa (tray na barra de menu, snap à grade, não aparecer no dock), identidade
visual Liquid Glass com vibrancy nativo do macOS e, ao final, empacotamento em `.dmg`.

## Plano (etapas incrementais, commit por etapa)

- **Etapa 0 — Recuperar ambiente:** remover artefatos do pnpm, reinstalar com npm para
  baixar o binário do Electron, validar que o widget abre. Ignorar `pnpm-*.yaml`.
- **Etapa 1 — Integração PWA (causa raiz):** ajustar o AppleScript para enxergar a janela
  do PWA `Amazon Music.app`; validar leitura de `mediaSession`/áudio e controle de volume.
- **Etapa 2 — Widget de mesa:** tray na barra de menu, nível de janela clicável "na mesa",
  snap à grade com posição persistida.
- **Etapa 3 — Liquid Glass nativo:** `vibrancy` do BrowserWindow + ajuste de CSS.
- **Etapa 4 — Persistência:** posição da janela e volume entre execuções.
- **Etapa 6 — Empacotamento `.dmg`:** `electron-builder`, target dmg, ícone `.icns`.
  (Sem Apple Developer ID → app não assinado/notarizado; contorno de Gatekeeper para uso pessoal.)

## Decisões aprovadas

- Corrigir/evoluir o código existente em vez de reconstruir do zero (fundação Electron +
  AppleScript é a única via viável sem API oficial do Amazon Music).
- Escopo: reconstrução completa de features.
- Visual: vibrancy nativo do macOS (Tahoe / macOS 26.5.1).
- Entregável final: app empacotado em `.dmg`.

## Contexto técnico confirmado

- macOS 26.5.1 (Tahoe) → Liquid Glass nativo disponível.
- Amazon Music instalado como **PWA standalone** (`~/Applications/Chrome Apps.localized/Amazon Music.app`).
- Causa de "não abrir mais": pnpm ignorou o build script do Electron (`ERR_PNPM_IGNORED_BUILDS`),
  então o binário (`node_modules/electron/dist/`) nunca foi baixado.

## Resumo do que foi feito

- **Etapa 0 concluída.** Ambiente Electron recuperado:
  - Removidos `pnpm-lock.yaml`, `pnpm-workspace.yaml` e a `node_modules` corrompida pelo pnpm.
  - Reinstalação via `npm install`. O postinstall do Electron 31 não completou sob Node v26
    (saía 0 sem extrair o binário — incompatibilidade do instalador de 2024 com o Node novo).
  - Binário extraído manualmente do cache (`~/Library/Caches/electron/.../electron-v31.7.7-darwin-arm64.zip`)
    e `path.txt` recriado. `electron --version` → `v31.7.7`.
  - Smoke test: `npm start` sobe o widget sem crash (processo principal + renderer vivos);
    janela confirmada visualmente pelo usuário.
  - `.gitignore` passa a ignorar `pnpm-*.yaml` para evitar reincidência.
- Etapas 1–6 não iniciadas — migradas para `PENDENCIAS.md`.

## Decisões técnicas relevantes

- Projeto usa **npm**, não pnpm (pnpm ignora build scripts e quebra o binário do Electron).
- Quando o postinstall do Electron falhar silenciosamente sob Node novo, a recuperação é
  extrair o zip do cache para `node_modules/electron/dist/` e escrever `path.txt` com
  `Electron.app/Contents/MacOS/Electron`.
